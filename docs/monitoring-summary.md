# Monitoring Summary

## Overview

이 문서는 AWS EKS 기반 서비스 운영 환경에서 적용한 모니터링 방식과 그 이유를 요약한 문서입니다.

이 프로젝트에서는 운영 초기에 모든 관측성 스택을 한 번에 구성하기보다, 실제 서비스 운영에 필요한 수준의 로그 및 메트릭 수집을 우선 확보하는 방향으로 접근했습니다.

핵심 목표는 다음과 같았습니다.

- 클러스터와 애플리케이션 상태를 기본적으로 확인할 수 있는 환경 확보
- 운영 시 필요한 로그와 지표 수집
- 장애 대응과 점검에 필요한 최소한의 기반 마련
- 운영 복잡도를 과도하게 늘리지 않는 현실적인 선택

---

## Monitoring Approach

초기 운영 단계에서는 CloudWatch Logs / Metrics를 중심으로 모니터링을 구성했습니다.

### 수집 대상 예시

- EKS 관련 로그
- 애플리케이션 로그
- 기본 인프라 메트릭
- 클러스터 운영 상태

### 이유

- AWS 관리형 서비스 기반으로 빠르게 적용 가능
- 별도 모니터링 스택 운영 부담이 적음
- 실제 운영 초기 단계에서 가장 현실적인 선택지
- 클러스터와 애플리케이션 상태를 우선 확인하는 데 충분한 기반 제공

---

## What Was Covered

### 1. Cluster and Platform Visibility

기본적인 클러스터 상태와 운영 지표를 확인할 수 있는 수준의 메트릭 수집을 목표로 했습니다.

예시:

- 클러스터 운영 상태 확인
- 노드 / 리소스 상태 확인
- 기본적인 로그 추적 기반 확보

### 2. Application Logging

애플리케이션 동작과 장애 원인 분석을 위해 로그 수집 기반을 마련했습니다.

예시:

- 애플리케이션 실행 로그
- 오류 로그
- 운영 중 점검에 필요한 기본 로그

### 3. Operational Monitoring

운영자가 상태를 점검할 수 있는 최소한의 모니터링 환경을 구성했습니다.

예시:

- 장애 상황 시 로그 추적
- 서비스 상태 점검
- 리소스 사용량 기반 운영 판단

---

## Why CloudWatch First

이 프로젝트에서는 Prometheus, Grafana, Loki, Promtail 같은 확장형 스택보다 CloudWatch를 먼저 사용했습니다.

### 이유

- AWS 환경과의 통합이 쉬움
- 빠른 도입 가능
- 별도 관리 포인트 감소
- 초기 운영 복잡도 완화
- 서비스 운영 안정화에 집중 가능

### 실무적인 의미

운영 초기에는 “가장 이상적인 구성”보다 “실제로 빠르게 운영 가능한 구성”이 더 중요할 수 있습니다.  
CloudWatch는 그런 점에서 합리적인 첫 선택이었습니다.

---

## Limitations

물론 CloudWatch 중심 운영만으로 모든 관측성을 해결할 수 있는 것은 아닙니다.

### 한계

- 대시보드 고도화 한계
- 세밀한 로그 분석 구조의 제한
- 장기적으로는 확장형 관측성 스택 필요
- Alerting 및 통합 모니터링 체계는 추가 정교화 필요

---

## Future Improvements

향후 보완하고 싶은 항목은 다음과 같습니다.

- Prometheus 기반 메트릭 수집 고도화
- Grafana 기반 시각화 고도화
- Loki / Promtail 기반 로그 분석 구조 확장
- Alerting 정책 정교화
- 운영용 Dashboard 표준화

---

## Trade-offs

이 구조는 확장형 관측성 스택보다 단순하지만, 운영 초기에는 충분히 실용적이었습니다.

### 감수한 점

- 세밀한 분석 기능 일부 제한
- 확장형 Dashboard 구성 부족
- 통합 Alerting 체계 미흡

### 대신 얻은 것

- 빠른 적용 가능
- 운영 복잡도 감소
- AWS 환경과 자연스러운 통합
- 서비스 운영 초기에 필요한 수준의 가시성 확보

---

## Outcome

CloudWatch 중심 모니터링 구성을 통해 다음과 같은 기반을 확보했습니다.

- 클러스터 및 애플리케이션 로그 수집 기반 확보
- 운영 상태 점검을 위한 기본 메트릭 수집 환경 마련
- 초기 운영 단계에 적합한 모니터링 체계 확보
- 향후 Prometheus / Grafana / Loki 기반 확장을 위한 출발점 마련
